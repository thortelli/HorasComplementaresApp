Sobre o sistema para Gerenciamento de Horas Complementares

Este site foi desenvolvido como trabalho de conclusão de curso do curso de Análise e Desenvolvimento de Software da Instituição de Ensino Instituito Federal de São Paulo - IFSP, no campus Guarulhos.

Visite o site da nossa instituiçao de ensino, no IFSP Guarulhos.

Porque um site para gerenciar as Horas Complementares?

As horas complementares, na grande maioria das instituições de ensino, são obrigatórias e um processo com muitas interações e ajustes. Com este comportamento um processo manual se torna extremamente oneroso e lento. Informatizar o prcoesso de envio e revisão de arquivos gera um grande ganho em tempo, seguraça e agilidade para ambos os lados envolvidos.

Do que é feito?

O Front-End foi desenvolvido em Angular e o back-end através do Asp.net Core Web API, utilizando as seguintes funcionalidades:

  • Asp.net Core: O Asp. net Core é uma das mais recentes ferramentas da Microsoft e possui inumeras funcionalidades e sub ferramentas complexas e de grande capacidade. Este projeto faz uso de algumas delas como o Entity Framework que se "responsabiliza" por gerenciar os usuários e suas credenciais.

  • Microsoft ASP.NET Core Identity : Utiliza a biblioteca da Microsoft para gerenciar os usuários e suas funções, além de ajudar a implementar estes usuários em repositorios gerenicos.

  • Angular: O agular é um framework para renderização da interface de aplicativos e paginas web. Ele é capaz de garantir vários beneficios como geração de código, templates, animações complexas, integração com varias plataformas, etc...

    Neste projeto o Algular faz uso do BootStrap 4 para os elementos visuais da págia o que garante uma boa performace para a renderização destas paginas.

  • NodeJS:

  • Visual Studio Code: O Visual Studio é uma interface de desenvolvimento que garante várias ferramentas para desenvolver um código estruturado e bem organizado, assim possibilitando um projeto robusto e de alta qualidade.

  • Swagger: O Swagger é uma ferramenta para facilitar o trabalho com APIs oferecendo alguns recursos como sumarizar as APIs do sistema para que o desenvolvedor não tenha que rever seus mapeamentos à todo momento ou relembrar quais parametros determinado serviço demanda, tudo será cotrolado pelo Swagger aumentando assim a produtividade do projeto.

  • SQL Server: O SQL Server é um sistema gerenciador de banco de dados capaz de lidar com grandes volumes de dados e estruturas complexas de armazenamento e excede em muito a capacidade necessária para lidar com este projeto, o que o torna uma ferramenta valida a ser mantida conforme o sistema pode vir a crescer.
